🎨 扩散模型原理

一句话

扩散模型 = 学会预测噪声。训练时给图加噪声让模型学预测，生成时从纯噪声开始一步步减掉噪声得到图片。

核心逻辑

干净图 = 噪声图 - 噪声

模型不画画，模型"擦玻璃"。它告诉你脏东西（噪声）在哪，你擦掉，画就出来了。

训练过程

for 每张训练图片:
    1. 随机选一个时间步 t（比如第 500 步）
    2. 给这张图加 t 步的噪声 → 得到模糊图
    3. 把模糊图 + 时间步 t 喂给模型
    4. 模型预测"这张图里有什么噪声"
    5. 用真实噪声和预测噪声算 loss
    6. 梯度下降，更新模型参数

生成过程（出图时）

步骤1：纯噪声 → 喂给模型 → 预测噪声 → 减掉 → 图A
步骤2：图A    → 喂给模型 → 预测噪声 → 减掉 → 图B
步骤3：图B    → 喂给模型 → 预测噪声 → 减掉 → 图C
...重复 30 次（steps=30）...
步骤30：→ 最终清晰图片

⚠️ 每一步都要调用模型。steps=30 就是跑 30 次模型，所以越多越慢。

UNet vs Transformer

UNet（SD 1.5 / SDXL 用的）

核心操作：卷积 — 用小窗口（3×3）在图上滑动，每次只看局部一小块。

结构：先压缩（看大局）→ 最小处做判断 → 再放大（还原细节）。形状像字母 U。

天花板原因：结构固定，加参数的方式有限，改一处其他都要跟着改。

Transformer（FLUX / SD3 / Wan 用的）

核心操作：注意力（Attention） — 每个位置都能直接看到所有其他位置。

结构：把图切成小块（patch），然后重复堆叠相同的 Attention 层。

优势：加层就能变强（scaling law），不需要"先压后放"的弯路。

代价：计算量大（O(n²)），更吃显存。

对比UNetTransformer
核心操作卷积（看局部）注意力（看全局）
结构固定的 U 形重复堆叠积木块
扩展性差（改不动）好（加层就行）
代表模型SD 1.5, SDXLFLUX, SD3, Wan
参数上限~3B 就到头可以堆到 100B+

显存与 GPU

显存 = GPU 的工作台。模型必须加载到显存里才能算。

GPU 快是因为能同时算很多东西（并行计算），像 10000 个人同时工作。

OOM = 显存不够 → 减 batch size / 用 fp16 / 量化

Mac 统一内存架构

CPU + GPU + 内存在同一块芯片上，共享内存。GPU 能用全部内存，但计算速度比 NVIDIA 慢很多。

nn.Module（定义网络）

固定套路：__init__ 里放层，forward 里定义数据怎么走。

class MyNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(784, 128)  # 784进，128出
        self.layer2 = nn.Linear(128, 10)   # 128进，10出

    def forward(self, x):
        x = self.layer1(x)      # 过第一层
        x = torch.relu(x)       # 激活
        x = self.layer2(x)      # 过第二层
        return x

# model(data) 自动调用 forward
output = model(data)

训练循环（核心五行）

训练本质 = 梯度下降 = 不断降低 loss

optimizer.zero_grad()               # 清空上轮梯度
output = model(data)                # forward（做题）
loss = loss_fn(output, label)       # 算 loss（对答案）
loss.backward()                     # 算梯度（知道哪里错）
optimizer.step()                    # 更新参数（改正）
角色做什么
model做预测的
loss_fn衡量预测和正确答案差多远
optimizer根据 loss 去调整 model 的参数

重复几十万次后模型就学会了。

🎯 LoRA 训练

LoRA 是什么

LoRA = 给模型加一个"小插件"，只训练这个插件（几十 MB），不动原模型（几 GB）。

效果：让基础模型学会特定风格/角色/概念。

训练流程

1. 准备 15-30 张目标风格图片

2. 每张图写一个标注（caption），包含触发词

3. 运行训练脚本，等 10-20 分钟

4. 得到 LoRA 文件，加载后用触发词出图

核心参数（调这三个就够）

参数作用调大了调小了
LORA_RANKLoRA 大小学得多但容易过拟合学得少但泛化好
LEARNING_RATE学习速度学得快但可能崩学得慢但稳定
TRAIN_STEPS训练多久学得充分但可能过拟合学得不够

不同场景推荐起点

场景ranklrsteps图片数
学画风161e-41000-150015-30
学人脸85e-5800-120010-20
学物体/IP161e-41000-200015-25
学构图321e-41500-250020-40

过拟合判断

表现：生成的图和训练图几乎一模一样，换 prompt 也不变。

解决：减 rank 或减 steps。

代码不用记，会改这些就行

每次训练新 LoRA 只需要改：

• 训练图片（换到 dataset 文件夹）

• 标注文本（每张图的 .txt 文件）

• 触发词（标注里的固定前缀）

• 三个参数（rank / lr / steps）

偶尔需要改代码的场景

• 图片格式混着（改文件过滤逻辑）

• 想中途生成样图看效果（加几行生图代码）

• 标注格式不同（改读文件的方式）

• 换模型架构（让 AI 重新生成脚本）

→ 这些都让 AI 帮你改，你负责审

关键概念

UNet = 预测噪声的核心网络 = LoRA 加在它上面

VAE = 图片编解码器（潜空间 ↔ 像素）

CLIP = 理解 prompt 文字的组件

Scheduler = 控制每步去多少噪声的调度器

Civitai = SD 生态最大的模型/LoRA 分享社区

• ComfyUI = 创作工具（试效果、调参、搭工作流）

• Python = 工程工具（训练、批量生成、部署 API）

• 两者不冲突，日常出图用 ComfyUI，做工程用 Python
